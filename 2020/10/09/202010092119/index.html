Python语法

1. 前言

这是我在学习《利用Python进行数据分析》第二章的记录。

2. Python解释器

Python有官方的解释器，可以在命令行交互，也可以执行。

IPython是一个强化的Python解释器，它和Jupyter notebooks语法一样，可以在解释器中用%run命令执行文件中的代码，并且结束之后可以与结果交互。而且Ipython还可以执行很多bash中命令。可以当成一个shell来用。

3. IPython基础

IPython解释器直接输出会组织成更舒适的格式，称为pretty-printed，如果是print函数，则会保持原样，可读性较差。并且，IPython还支持执行任意代码块和整段Python脚本。

Jupyter Notebook是交互式文档，和IPython shell几乎所有的命令和工具都可以通用。

Tab补全

Tab补全是一个很好用的功能，Jupyter需要安装插件才可以，IPython中直接用。IPython会隐藏下划线开头的方法和属性。

自省

在变量前后使用问号?可以显示对象的信息。如果对象是一个函数或实例方法，定义过的文档字符串，也会显示出信息。

使用一个问号会返回注释里的信息，如果使用两个问号，则会显示函数的源码。

?还有一个用途，像命令行一样搜索IPython的命名空间。字符与通配符结合可以匹配所有的名字。

4. %run命令

在IPython解释器中，%run就像python一样。这段脚本运行之后，文件中所有定义的变量，都可以在IPython shell中随后访问。-i参数，可以加入命令行参数。

%load指令，将脚本导入到代码格中，对于函数，并不会直接运行输出结果。发现%run其实也不会输出结果。

Ctrl-C可以导致Python程序停止。

%paste和cpaste会直接运行剪贴板

5. 键盘快捷键和魔术函数

键盘快捷键的功能很常规，自行查找使用。魔术命令特点是%，可以看作IPython中有粘性的命令行，一些情况下会很有帮助。

matplotlib

通过魔术命令运行matplotlib。

6. Python语法基础

  • Python使用缩进而不是括号。分号可以给同一行的语句划分。
  • Python中赋值不是复制，而是同一个对象。即赋值后两个变量实际内容都是一样的。
  • Python虽然没有声明变量，但是是强类型的语言。可以用isinstance判断对象是某个类型的实例。isinstance(a, (int, float))
  • 鸭子类型，判断是否迭代器和某种类型实例。有一个__iter__魔术方法。
  • 导入模块。
  • 运算。is方法可以判断两个对象是否相同。a is b。这和==不同，is是指向同一对象。
  • 可变与不可变对象。字符串和元组是不可变的。尽量避免副作用，进能量避免修改对象。
  • 数值类型。int任意大，float双精度，也可以用科学计数法表示。/会得到浮点数, //C风格除法。
  • 字符串。有换行符的字符串可以用三引号。c.count('\n')可以计算行数。反斜杠用来转义，’r’可以说明字符就是本身。字符串的格式化用{2:.2f}或s,d这样的形式。字节文本前面加上一个b。
  • 日期和时间，内建的datetime模块提供了datetime, date, time。
  • break, continue, pass。
  • range产生迭代器，三个参数是起点终点和步进。
  • 三元表达式value = true if condition else false。

7. 数据结构和序列

元组

元组是固定长度，不可改变的序列对象。最简单方式是用逗号分隔一列值。

可以用方括号访问，但不能修改。如果元组中某个对象可变，可以在原位修改。可以理解为指针。可以串联和复制。

对象本身并没有复制，只是引用了它。

拆分

注意拆分时候是复制。

利用元组赋值可以起到拆分的作用，替换变量名字很容易。

用*rest可以实现抓取任意长度列表的位置参数。

tuple方法

count比较有用。

列表

与元组相比，列表的长度可变、内容可以被修改。

添加和删除元素

list.append('ad')。

list.insert(1, 'red')。insert的逆运算是pop，它移除指定位置的元素；而remove移除某个值的第一个对象。

in可以用来判断列表是否包含某个值。

切片

[start:end:step]注意可以有三个参数。

一些函数
序列函数

enumerate可以返回(i, value)的元组序列。

sorted函数

可以从任意序列的元素返回一个新的排好序的列表。

zip函数

zip函数可以将多个列表、元组或其它序列成对组合成一个元组列表。元素的个数取决于最短的序列。最常见用法是同时迭代多个序列，结合enumerate使用。

zip还可以实现压缩序列的功能。

reversed函数

可以从后向前迭代一个序列。

字典

字典是键值对的大小可变集合。可以用del或pop方法（返回值的同时删除键）删除值。

迭代器方法keys和values，虽然键值对没有顺序，但是这两个方法可以用相同的顺序输出键和值。

update方法可以将一个字典与另一个融合。

序列创建字典
  • zip循环；
  • dict(zip(a, b))函数。
默认值

get默认会返回None。

by_letter.setdefault(letter, []).append(word)可以实现通过首字母，将一个列表中的单词分类。

有效的键类型

字典的值可以是任意Python对象，而键通常是不可变的标量类型。这叫做可哈希性。要用列表当作键，一种方法是将列表转化成元组，只要内部元素可以被哈希，它也可以被哈希。

集合

集合是无序的不可重复的元素的集合。

常用的集合方法
不可变

列表不可以，必须转换成元组。

列表、集合和字典推导式

在里面嵌套一级或者多级循环。

8. 函数

函数是Python最重要的代码组织和复用手段。函数使用def关键字声明，用return关键字返回值。可以写多条return语句，如果没有遇到任何一条return语句，则返回None。函数可以有一些位置参数和关键字参数。关键字参数必须位于位置参数之后。

命名空间、作用域，和局部函数

变量的作用域有两种，全局和局部。Python中描述变量作用域的名称，叫做命名空间。函数中复制的变量默认都是被分配到局部命名空间。座椅，下面情形必须给a用global关键字声明。

nonlocal可以使用外层的变量。

不要频繁使用global关键字，如果使用了很多，可能就说明要来点面向对象编程了。

返回多个值

return a, b, c 其实返回的是元组，我们可以拆分他。我们还可以返回字典：return {'a':a, 'b':b, 'c':c}

函数也是对象

Python函数都是对象，所以在其他语言中难表达的一些思想在Python中会简单很多。比如字符串去掉一些无聊的标志：value.strip().我们看到字符串作为对象，用它的方法处理很方便。

函数列表

我们还可以将不同的处理的函数做成一个列表，然后不断用这个列表中的函数对值进行处理。

函数作为参数

比如map函数，我们可以在一组数据上应用另一个函数。

匿名函数

Python支持一种被称为匿名的、或lambda函数。它仅由单条语句组成，该语句的结果就是返回值。通过lambda关键字定义。

根据个字符串不同字母数量对其排序：

柯里化：部分参数应用

柯里化，指的是通过“部分参数应用”从现有函数派生出新函数。

生成器

能以一种一致的方式对序列进行迭代时Python的一个重要特点。通过迭代器协议的方式实现的。

比如for key in some_dict:，解释其首先会尝试iter(some_dict)创建一个迭代器。

生成器时构造新的可迭代对象的一种简单方式。一般函数执行之后返回单个值，生成器一个一个地返回。

生成器表达式

创建方式为：把列表推导式两端的方括号改成圆括号。生成器表达式可以取代列表推导式，作为函数参数。

itertools模块

itertools模块中有一组用于许多常见数据算法的生成器。

错误和异常处理

错误和异常处理与分支结构不同，分支是二选一，错误和异常处理是先走一，一不行才走二。

异常类型

except只在失败时运行，else只在成功时运行，finally无论如何都运行。

9. 文件和操作系统

我们可以用内置的open函数以及一个相对或绝对的文件路径打开文件以便读写。默认是以制度模式打开的，然后我们可以像对列表那样对文件句柄f处理。注意，文件中去除的行都带有完整的行结束符EOL。

打开后一定要关闭，f.close()，这样可以返回操作系统资源。

with语句可以更容易地清理打开的文件。

模式 说明
r 只读模式
w 只写模式，会创建新文件
a 附加到现有文件，也会创建新文件
r+ 读写模式
b 用于二进制文件
U 通用换行模式

对于可读文件, read(), seek(), tell(), readlines()会从文件返回字符。

写入，可以用write()或者writelines()方法。

还有flush()和closed()等文件方法。

文件的字节和Unicode

Python文件默认操作是“文本模式”，即需要处理Unicode，它与“二进制模式”相对。

UTF-8是长度可变的Unicode编码，所以有时一个字符串用二进制形式解码并不都能解出字符。

注意不要再二进制模式中使用seek，可能会出错。
