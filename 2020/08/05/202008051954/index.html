Python pdb调试速查记录

1. 介绍

The module pdb defines an interactive source code debugger for Python programs. It supports setting (conditional) breakpoints and single stepping at the source line level, inspection of stack frames, source code listing, and evaluation of arbitrary Python code in the context of any stack frame. It also supports post-mortem debugging and can be called under program control.

pdb就是Python自带的一个简陋的命令行调试功能，而ipdb就是IPython解释器提供的更强大的调试功能。

2. 单步执行代码

  • h (help) 帮助
  • w (where) 打印当前执行堆栈
  • d (down) 执行跳转到再当前堆栈的深一层
  • u (up) 执行跳转到当前堆栈上一层
  • b (break) 添加断点
    • b 列出当前所有断点，和断点执行到统计次数
    • b line_no 当前脚本的line_no行添加断点
    • b filename:line_num 某脚本某行添加断点
    • b function 在函数第一跳语句处添加断点
  • tbreak (temporary break) 临时断电
  • cl (clear) 清除断点
    • cl 清除所有断点
    • cl bpnumber1 bpnumber2... 清除断点号为bpnumber1,bpnumber2…的断点
    • cl lineno 清除当前脚本lineno行的断点
    • cl filename:line_no 清除脚本filename的line_no行的断点
  • disable 停用断点
  • enable 激活断点
  • s (step) 执行下一条命令
  • n (next) 执行下一条语句
  • r (return) 执行当前运行函数到结束
  • c (continue) 继续执行，直到遇到下一个断点
  • l (list) 列出源码
    • l 列出当前执行语句周围11条代码
    • l first 列出first行周围11条代码
    • l first second 列出first—second范围的代码，如果second<first，second将被解析为行数
  • a (args) 列出当前执行函数的函数
  • p expression (print) 输出值
  • pp expression (print) 好看一点的输出
  • run 重新启动debug
  • q (quit) 退出debug
  • j lineno (jump) 设置下条执行的语句函数
  • unt (until) 执行到下一行(跳出循环)，或者当前堆栈结束
  • condition bpnumber condition 给断电设置条件，参数condition返回True时有效

3. 注意

  • 直接输入Enter，会执行下一条命令
  • 输入PDB不认识的命令，会当作Python语句执行

4. 导入执行

在import pdb之后，直接在代码里需要调试的地方放一个pdb.set_trace().
