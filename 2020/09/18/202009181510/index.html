onda环境下的包管理

1. conda和pip区别

conda和pip是两个常用的包管理工具，应用时我们通常将两者结合起来使用。

两者的主要区别：

简单说，Pip安装Python软件包，conda安装软件包，其中可能包含以任何语言编写的软件；Pip需要已经安装了编译器，而conda不需要任何编译器；Conda能够创建隔离的环境；Conda会解决包依赖的问题。

2. 比较好的实践

当找到一篇新的论文，打算复现的时候。

  1. 利用conda create -n new_env创建新环境；(或许可以利用--clone old_env)
  2. 利用conda install pip确保pip安装的位置在本环境;
  3. pip install -r requirements.txt，安装。

3. 如果pip安装卡住

  1. 利用pip install package先找到下载的文件的位置；
  2. 手动下载文件；
  3. tar -xvf test.tar.gz -C dir解压；
  4. 打开解压的文件，运行python setup.py install;
  5. 将与setup.py同目录的文件夹移到环境中的包列表。

4. conda环境中的包的位置

可以先用which python查一下python位置确定env的位置，然后找到envs/super/lib/python3.8/site-packages/scikit-learn/即可。
